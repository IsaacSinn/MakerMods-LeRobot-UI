# 自动校准测试说明

本文档说明如何运行飞特舵机**自动校准**：一条命令完成展开避干涉 + 全部校准 + 回中折叠的完整流程。

---

## 一、完整流程概览

| 阶段 | 内容 | 说明 |
| --- | --- | --- |
| 阶段0 | 初始化 | 停止所有舵机，Lock=1，配置 PID/Acceleration/限位等，上力矩 |
| 阶段2 | 展开 2-4 号并记录参考位置 | 先抬 wrist_flex(4)→shoulder_lift(2)→elbow_flex(3)，记录展开方向与参考位置，再收回 2、3 号 |
| 校准 2、3 号 | 多舵机同时校准 | shoulder_lift、elbow_flex 一次测量写回，折叠 2、3 号 |
| 阶段3 | 校准 4、5、6 号 | 抬肘 80° 避障后，多舵机同时校准 wrist_roll、gripper、wrist_flex；再折叠 3 号、展开 4 号、折叠 gripper |
| 阶段4 | 校准 1 号并回中 | 校准 shoulder_pan，写回并回中 |
| 结果与释放 | 打印结果、折叠、释放 | 打印各关节 min/max/偏移量，全臂折叠，按回车释放扭矩 |
| 持久化（可选） | 与手动校准相同方案 | 仅 `--save` 时：写入舵机 EEPROM，并保存到 `.../calibration/robots/so_follower/<robot_id>.json`（draccus 格式，机械臂启动时加载） |

使用 `--unfold-only` 时只执行阶段0 与 阶段2 的展开（仅臂展开，不校准），便于单独调试展开逻辑。

---

## 二、如何调用

### 2.1 环境准备

使用 CLI 命令前需先安装包：

```bash
pip install -e .
```

安装后即可使用 `lerobot-measure-feetech-ranges` 命令。如果未安装或不想安装，可以直接用 Python 调用：

```bash
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23
```

以下示例统一使用 Python 调用方式，替换为 `lerobot-measure-feetech-ranges` 效果相同。

### 2.2 基本用法

```bash
# 完整流程（展开+校准），不持久化
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23

# 完整流程 + 持久化（EEPROM + 与手动校准相同的本地文件）
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --save

# 指定机器人 id，与机械臂 config.id 一致时启动可直接加载
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --save --robot-id bimanual_follower_left

# 跳过展开（已展开或无需展开）
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --unfold-angle 0

# 只校准单个舵机（跳过展开）
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --motor shoulder_pan

# 自定义展开角度
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --unfold-angle 30

# 仅调试臂展开（只做阶段0 初始化 + 阶段2 展开，不校准）
python src/lerobot/scripts/lerobot_measure_feetech_ranges.py --port COM23 --unfold-only
```

### 2.3 舵机名称（ID 1～6）

- `shoulder_pan` (1)
- `shoulder_lift` (2)
- `elbow_flex` (3)
- `wrist_flex` (4)
- `wrist_roll` (5)
- `gripper` (6)

### 2.4 参数列表

**校准参数：**

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `--port` | 串口路径（必填） | 无 |
| `--motor` | 仅校准该舵机（跳过展开） | 全部 |
| `--velocity-limit` | 校准探测限位速度（恒速模式 Goal_Velocity） | 1000 |
| `--timeout` | 单方向限位等待超时（秒） | 20.0 |

**展开参数：**

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `--unfold-only` | 仅执行臂展开（阶段0 + 阶段2），不校准，用于调试展开逻辑 | 关闭 |
| `--unfold-angle` | 展开角度（度），设为 0 跳过展开 | 45.0 |
| `--unfold-timeout` | 展开运动超时（秒），到达或堵转会提前结束 | 6.0 |

**输出参数（与手动校准相同路径与格式）：**

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `--save` | 写入舵机 EEPROM，并保存到与手动校准相同的本地路径（draccus 格式） | 不写入 |
| `--robot-id` | 保存文件名对应的机器人 id，路径为 `.../calibration/robots/so_follower/<robot_id>.json`，需与机械臂启动时 config.id 一致 | default |

## 三、详细流程

### 3.1 阶段0：初始化

- 先用 `sync_write` 广播清除残留 Overload（不需要回包）
- 重新连接并验证所有舵机
- 停止所有 6 个舵机（`Torque_Enable=0`）
- 设置 `Lock=1` 锁定 EEPROM
- 配置寄存器（写后读回校验，异常时打印设置值/读到值）：

| 寄存器 | 值 | 说明 |
| --- | --- | --- |
| Return_Delay_Time | 0 | 减少通信延迟 |
| Operating_Mode | 0 | 伺服模式 |
| Max_Torque_Limit | 1000 | 最大扭矩 |
| Torque_Limit | 600 | 运行扭矩限制 |
| Acceleration | 50 | 加速度（与 calibration_defaults 一致） |
| P_Coefficient | 16 | 位置环 P 系数（防抖动） |
| I_Coefficient | 0 | 位置环 I 系数 |
| D_Coefficient | 32 | 位置环 D 系数 |
| Min/Max_Position_Limit | 0 / 4095 | 临时全量程，校准后再写回 |
| Homing_Offset | 0 | 临时清零 |
| Torque_Enable | 1 | 上力矩 |

（gripper 等特殊保护由机械臂 `configure` 时设置，本脚本阶段0 不单独设置。）

### 3.2 阶段2：展开 2-4 号并记录参考位置

1. 按 4→2→3（wrist_flex → shoulder_lift → elbow_flex）抬臂：wrist_flex 80°、shoulder_lift 15°、elbow_flex 30°，并记录各关节展开方向（forward/reverse）及参考位置 `(Present_Position + Homing_Offset) % 4096`，供后续校准时选弧。
2. 折叠 shoulder_lift、elbow_flex 回中。
3. 校准 2、3 号：多舵机同时执行 `measure_ranges_of_motion_multi`（先抬起的反方向为首次旋转方向），写回 Homing_Offset 与限位，2、3 号回中或保持后折叠 2、3 号。

展开单关节时：校正中位（`Torque_Enable=128`）、尝试正/反方向 `Goal_Position = 2048 ± 目标步数`，堵转或超时则回中换向。

### 3.3 阶段3：校准 4、5、6 号舵机

1. 抬肘 80° 避障。
2. 多舵机同时校准 wrist_roll(5)、gripper(6)、wrist_flex(4)：一次 `measure_ranges_of_motion_multi`，写回后 wrist_roll 保持当前位置，其余回中。
3. 一次 _fold_arm：elbow_flex 折叠、wrist_flex 展开、gripper 折叠。

### 3.4 阶段4：校准 1 号 shoulder_pan 并回中

1. 对 shoulder_pan 执行多舵机校准（单电机）流程，写回 Homing_Offset 与限位。
2. 回中（Goal_Position=中位）。

### 3.5 测量与写回（measure_ranges_of_motion_multi）

核心逻辑在 `auto_calibration.py`：对每个电机恒速模式探测 CW/CCW 限位，用参考位置（若有）选弧、算中值与 Homing_Offset，再写回 Homing_Offset 与 Min/Max 限位。wrist_roll 写回后保持当前位置，其余回中。多舵机时一次调用同时测量多个电机。

### 3.6 运动控制策略

伺服模式下**不使用 Goal_Velocity 控制速度**，而是通过 PID 参数和 Acceleration 控制运动行为：

- `P_Coefficient=16`：较低的 P 值使响应更柔和，防止抖动
- `Acceleration=50`：平滑加速（与 calibration_defaults 一致）
- 只需写 `Goal_Position` 即可驱动舵机移动

恒速模式（校准探测限位时使用）仍使用 `Goal_Velocity` 控制转速。

## 四、堵转检测

所有堵转检测统一使用 **Status 寄存器（地址 65，0x41）的 BIT5**：

- BIT5=1 表示**过载（堵转）**
- 校准阶段：连续 3 次检测到 BIT5=1 才判定堵转（避免误判）
- 展开阶段：检测到 BIT5=1 立即判定堵转

通信异常（RuntimeError/ConnectionError）也视为堵转信号。

---

## 五、0/4095 回绕处理

单圈模式下位置值为 0-4095 循环。CW 和 CCW 极限把圆环切成两段弧，利用起始位置判断哪段弧是物理可活动范围：

- 弧 A = (pos_cw - pos_ccw) % 4096
- 弧 B = (pos_ccw - pos_cw) % 4096
- 起始位置在哪段弧内，那段弧就是物理范围
- 中值 = 物理范围弧的起点 + 范围/2

## 六、涉及的寄存器

| 寄存器 | 地址 | 用途 |
| --- | --- | --- |
| Return_Delay_Time | 0x07 (7) | 应答延迟，设为 0 减少通信延迟 |
| Min_Position_Limit | 0x09 (9) | 最小角度限位 |
| Max_Position_Limit | 0x0B (11) | 最大角度限位 |
| Max_Torque_Limit | 0x10 (16) | 最大扭矩，初始化设为 1000 |
| Phase | 0x12 (18) | BIT4: 角度反馈模式，0=单圈(0-4095) |
| P_Coefficient | 0x15 (21) | 位置环 P 系数，设为 16（柔和响应） |
| D_Coefficient | 0x16 (22) | 位置环 D 系数，设为 32 |
| I_Coefficient | 0x17 (23) | 位置环 I 系数，设为 0 |
| Protection_Current | 0x1C (28) | 保护电流（gripper 由机械臂 configure 时设 250） |
| Homing_Offset | 0x1F (31) | 位置偏移，BIT11 符号位 |
| Operating_Mode | 0x21 (33) | 运行模式，0=伺服模式，1=恒速模式 |
| Overload_Torque | 0x24 (36) | 过载保护扭矩（gripper 由机械臂 configure 时设 25） |
| Torque_Enable | 0x28 (40) | 扭矩开关，128=当前位置设为 2048 |
| Acceleration | 0x29 (41) | 加速度，脚本阶段0 设为 50（与 calibration_defaults 一致） |
| Goal_Position | 0x2A (42) | 目标位置（伺服模式，写入自动使能扭矩） |
| Goal_Velocity | 0x2E (46) | 目标速度（仅恒速模式校准时使用） |
| Torque_Limit | 0x30 (48) | 转矩限制，脚本阶段0 设为 600 |
| Lock | 0x37 (55) | EEPROM 锁，1=锁定 |
| Present_Position | 0x38 (56) | 当前位置（只读） |
| Status | 0x41 (65) | 舵机状态（只读），BIT5=1 表示过载/堵转 |

### 7.2 异常情况

- 连接失败：打印错误，退出码 1
- 测量异常：打印错误，退出码 1
- Ctrl+C 中断：安全停止所有舵机，退出码 130

## 八、实现位置与保存路径

| 文件 | 说明 |
| --- | --- |
| `src/lerobot/scripts/lerobot_measure_feetech_ranges.py` | 主脚本（阶段0 初始化、展开、多舵机校准、持久化与手动校准同路径） |
| `src/lerobot/motors/feetech/auto_calibration.py` | 多舵机测量（`measure_ranges_of_motion_multi`）、展开单关节、WritePosEx、堵转检测等 |
| `src/lerobot/motors/feetech/feetech.py` | 总线与 FeetechCalibrationMixin，导出 `COMM_ERR` |
| `src/lerobot/motors/feetech/calibration_defaults.py` | 默认参数（速度、超时、PID、Acceleration、关节顺序等） |

**持久化路径（与手动校准一致）**：`HF_LEROBOT_CALIBRATION / "robots" / "so_follower" / "<robot_id>.json"`，默认即 `~/.cache/huggingface/lerobot/calibration/robots/so_follower/<robot_id>.json`。使用 draccus 格式（`dict[str, MotorCalibration]`），机械臂启动时通过 `config.id` 加载同 id 的该文件。
